# 数据同步策略

## 初始状态

* **空启动**: 应用程序安装时，本地 **Room** 数据库默认为**空**，以保持 APK 体积最小化（"极致轻量"）。

## 初始化工作流 ("空数据检测")

1. **App 启动**:
    * 主页面 (`HomeViewModel`) 检查数据库中的记录数。
2. **检测**:
    * **条件**: 如果 `count == 0`。
    * **UI 响应**: 显示醒目的“需要设置”或“下载数据”卡片。
3. **用户操作**:
    * 用户点击“下载”。
4. **下载与处理**:
    * **机制**: 使用 `WorkManager` 或前台 Service (Foreground Service)（不在主线程执行）。
    * **网络**: **Retrofit** 下载压缩的数据库/数据文件。
    * **存储**: 解析并批量插入 **Room**。
    * **完成**: 通知主页面刷新 UI（状态变更为“已填充”）。

## 云端更新

* **增量 vs 全量**: （需根据后端定义）支持高效更新。
* **超时**: 任何实时网络检查（如果未来实现）严格限制在 **2秒** 内。

## 线程处理

* 所有数据库写入必须在 `Dispatchers.IO` 上执行。
* 来电查询 (`CallScreeningService`) 应该使用 `RunBlocking`（需谨慎）或具有严格超时的专用 Main-Safe
  异步操作，如果未使用专用查找结构，最好使用优化的非阻塞 IO。
