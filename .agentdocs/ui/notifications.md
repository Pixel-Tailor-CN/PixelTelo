# 数据同步与通知策略

## 1. 数据库更新机制

APP 支持从云端下载离线数据库以更新本地 Room 数据库。

## 2. 通知交互规范

### 自动检查更新

* 设置页提供“自动检查更新”开关，默认关闭。
* 开启后使用 `WorkManager` 按用户配置间隔检查离线数据库版本，默认 24 小时。
* 检测到新版本时发送普通通知，通知点击后打开 Pixel Telo，由用户手动确认下载。
* 同一个数据库版本只提醒一次，避免周期性重复通知。
* Android 13+ 需要 `POST_NOTIFICATIONS` 权限；未授权时不允许开启自动检查更新，不影响手动更新与来电拦截。

### 下载过程中

* **Android 16+**: 使用 `LiveUpdateNotification` 形式展示下载进度（需调研具体 API 或使用最接近的系统特性）。
* **Android <16**: 使用标准的前台服务通知 (Foreground Service Notification) 展示下载进度条。

### 更新完成后

* 发送一条普通通知告知用户更新完成。
* **内容包含**:
    * 新数据库版本号
    * 数据库文件大小

## 3. 实现细节

* 使用 `WorkManager` (推荐 `Expedited Work`) 或前台服务来执行下载任务，确保在后台不被杀掉。
